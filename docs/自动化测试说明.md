# QuickTrans 自动化测试说明

## 一、环境与依赖

- **测试框架**: [Vitest](https://vitest.dev/)（与 Vite 同源，无需额外 Jest 配置）
- **运行环境**: Node（当前用例不依赖浏览器 DOM）
- **已用依赖**: `vitest`、`@vitest/coverage-v8`、`jsdom`、`@testing-library/react`、`@testing-library/jest-dom`、`@testing-library/dom`（后几项为后续组件测试预留）

请先安装依赖：

```bash
npm install
```

若 `npm install` 报错，可尝试：

- 使用 `npm ci`（需存在 `package-lock.json`）
- 或检查 Node 版本（建议 18+）、网络与 npm 源

## 二、如何执行测试

在项目根目录执行：

| 命令 | 说明 |
|------|------|
| `npm run test` | 监听模式：修改代码后自动重新跑测 |
| `npm run test:run` | 单次运行全部测试并退出 |
| `npm run test:coverage` | 单次运行并生成覆盖率报告（输出在 `coverage/`） |

首次运行或只跑一次验证时，建议：

```bash
npm run test:run
```

## 三、测试范围与约定

- **用例位置**: 与源码同目录，文件名形如 `*.test.ts` / `*.test.tsx`
- **当前覆盖**:
  - `src/converters/TimestampConverter.test.ts`：时间戳 ↔ 日期转换、时区
  - `src/converters/ContentDetector.test.ts`：选中内容类型检测（时间戳 / 日期 / URL）
  - `src/devtools/utils/timestampScanner.test.ts`：JSON/文本中的时间戳扫描
  - `src/devtools/utils/urlExtractor.test.ts`：文本/JSON/请求中的 URL 提取
- **未覆盖**: 依赖 `chrome.*` API、React 组件、content script 等，可在后续通过 mock Chrome / jsdom 或 E2E 补充

## 四、新增用例建议

1. 在对应模块旁新增 `xxx.test.ts`（或组件用 `xxx.test.tsx`）。
2. 使用 Vitest 的 `describe` / `it` / `expect`，例如：

```ts
import { describe, it, expect } from 'vitest';
import { YourModule } from './YourModule';

describe('YourModule', () => {
  it('应满足某行为', () => {
    expect(YourModule.doSomething()).toBe(expected);
  });
});
```

3. 运行 `npm run test` 或 `npm run test:run` 确认通过。

## 五、与手动测试的关系

- **自动化测试**（本文）：回归用，保证核心逻辑稳定，适合 CI。
- **手动功能测试**：见 [功能测试用例](./功能测试用例.md)，在真机/真实扩展环境下做 UI 与交互验证。

建议：提代码前跑一遍 `npm run test:run`，发版前再按《功能测试用例》做一次手动检查。
