# QuickTrans 功能测试用例（手动执行）

本文档为 **手动功能测试** 用例，供你在本地按步骤执行并勾选通过/不通过。非单元测试或自动化脚本。

---

## 前置条件

- [ ] Chrome 浏览器已安装
- [ ] 项目已构建：`npm run build`，并在 Chrome 扩展管理页加载 `dist` 目录
- [ ] 准备一个可访问的普通网页（如 https://www.example.com 或任意 HTTP/HTTPS 页面）用于测试内容脚本与 Popup

---

## 一、插件安装与基础入口

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 1.1 | 扩展加载 | 打开 `chrome://extensions`，确认 QuickTrans 已启用 | 列表中显示 QuickTrans，无报错 | ☐ |
| 1.2 | 扩展图标 | 查看浏览器工具栏 | 显示 QuickTrans 图标 | ☐ |
| 1.3 | Popup 打开 | 点击扩展图标 | 弹出 Popup 窗口，显示标题、工具/设置/功能导航 | ☐ |
| 1.4 | DevTools 入口 | 打开任意网页 → F12 打开开发者工具 → 顶部 Tab | 存在「QuickTrans」或「QuickTrans Network Panel」等 Tab | ☐ |

---

## 二、Popup - 工具页（时间戳 / 二维码入口）

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 2.1 | 时间戳入口 | 在 Popup 默认「工具」页，点击「时间戳转换」 | 当前标签页出现时间戳转换悬浮窗（可输入时间戳或日期） | ☐ |
| 2.2 | 二维码入口 | 在工具页点击「二维码生成」 | 当前标签页出现二维码生成悬浮窗（可输入链接生成二维码） | ☐ |
| 2.3 | 无标签页/特殊页 | 当前 Tab 为 `chrome://extensions` 或 `about:blank`，再点击时间戳/二维码 | 显示错误提示（如「无法在此页面使用」或「请打开普通网页」） | ☐ |
| 2.4 | 无活动标签 | 关闭所有普通网页仅留扩展页，点击工具 | 显示无法获取当前标签页或请打开网页类提示 | ☐ |

## 三、Popup - 设置页（参数配置）

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 3.1 | 打开设置 | Popup 中点击「设置」 | 进入设置页，显示参数列表（可为空） | ☐ |
| 3.2 | 添加参数 | 点击「添加参数」，输入 key 如 `token`，value 如 `abc123`，保存 | 列表中出现 `token=abc123` | ☐ |
| 3.3 | 编辑参数 | 点击某条参数的编辑按钮，修改 key 或 value 后保存 | 列表中显示更新后的值 | ☐ |
| 3.4 | 删除参数 | 点击某条参数的删除按钮 | 该条从列表中移除 | ☐ |
| 3.5 | 空 key 不可保存 | 添加时 key 留空，点击保存 | 保存按钮禁用或无法保存 | ☐ |

## 五、网页内 - 选中时间戳

在**普通网页**（如 example.com）的页面或控制台里准备一段可选中文本（如时间戳）。

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 5.1 | 10 位时间戳 | 选中 `1704067200`，点击出现的「转换时间」按钮 | 出现时间戳转换悬浮窗，显示对应日期时间 | ☐ |
| 5.2 | 13 位时间戳 | 选中 `1704067200000`，点击「转换时间」 | 悬浮窗显示正确日期时间 | ☐ |
| 5.3 | 空选打开面板 | 不选中任何内容，从 Popup 点击「时间戳转换」 | 仍打开时间戳面板，可手动输入时间戳或日期 | ☐ |
| 5.4 | 面板内输入转换 | 在面板输入 `1704067200000` 或一列日期时间 | 显示转换结果，无报错 | ☐ |
| 5.5 | 复制结果 | 在时间戳面板点击复制（若有） | 剪贴板得到对应时间或时间戳 | ☐ |

## 六、网页内 - 选中链接生成二维码

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 6.1 | 选中 URL | 在网页选中 `https://www.example.com`，点击「生成二维码」 | 出现二维码悬浮窗，显示该链接的二维码 | ☐ |

| 6.4 | 无链接时从 Popup 打开 | 不选中链接，从 Popup 点击「二维码生成」 | 打开二维码面板，可输入链接后生成 | ☐ |

## 七、右键菜单

在普通网页中操作。

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 7.1 | 转换时间戳 | 选中时间戳（如 `1704067200000`）→ 右键 | 菜单中有「转换时间戳」或类似项，点击后出现时间戳转换悬浮窗 | ☐ |
| 7.2 | 生成二维码 | 选中 URL（如 `https://example.com`）→ 右键 | 菜单中有「生成二维码」或类似项，点击后出现二维码悬浮窗 | ☐ |
| 7.3 | 识别二维码 | 在页面找到一张二维码图片 → 右键该图片 | 菜单中有「识别二维码」或类似项，点击后出现识别结果（链接/文本等） | ☐ |
| 7.4 | 非二维码图片 | 右键一张普通图片（非二维码） | 仍可看到「识别二维码」，识别结果可为空或提示非二维码 | ☐ |

## 八、DevTools - QuickTrans Network 面板

打开任意会产生 **fetch/XHR** 请求的网页（如带接口的前端项目），再打开 DevTools 并切到 QuickTrans（Network）面板。

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 8.1 | 请求列表 | 刷新页面或触发接口请求 | 列表中出现 fetch/xhr 请求，显示 URL、状态等 | ☐ |
| 8.2 | 选中请求 | 点击列表中某条请求 | 右侧详情区显示该请求的详情 | ☐ |
| 8.3 | 搜索 | 在搜索框输入 URL 关键词或部分路径 | 列表仅显示匹配的请求 | ☐ |
| 8.4 | 状态过滤 | 状态下拉选 200 / 404 / 500 | 列表仅显示对应状态码的请求 | ☐ |
| 8.5 | 暂停捕获 | 点击「暂停」 | 后续新请求不再加入列表（或界面有暂停态） | ☐ |
| 8.6 | 恢复捕获 | 再次点击（恢复） | 新请求重新被记录 | ☐ |
| 8.7 | 清空列表 | 点击「清空」并确认 | 列表清空，右侧详情区为空或提示选择请求 | ☐ |
| 8.8 | 请求数统计 | 查看标题旁请求数量 | 数字与当前列表条数一致 | ☐ |
| 8.9 | 语言切换 | 在面板内切换中/英文 | 标题、占位符、按钮等文案随语言切换 | ☐ |
| 8.10 | 快捷键 | 在面板内按 Cmd/Ctrl + F | 搜索框获得焦点 | ☐ |

## 九、DevTools - 请求详情（Response / Request / 时间戳 / URL / 二维码）

在 Network 面板选中一条**返回 JSON 且含时间戳或 URL** 的请求（若没有可先造一条或选已有接口）。

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
| 9.1 | Response 标签 | 选中请求后查看「Response」或「响应」 | 显示响应体（如 JSON 格式化） | ☐ |
| 9.2 | Request 标签 | 切换到「Request」/「请求」 | 显示请求体（若有） | ☐ |
| 9.3 | Headers 标签 | 切换到 Headers | 显示请求/响应头信息 | ☐ |
| 9.4 | 时间戳 Tab | 若响应中含时间戳，切换到「时间戳」/ Timestamps | 列出识别到的时间戳，可点击转换或查看 | ☐ |
| 9.5 | URL Tab | 切换到 URL/链接 相关 Tab | 列出提取到的 URL，可打开或复制 | ☐ |
| 9.6 | 二维码展示 | 若某 URL 支持生成二维码，在详情中查看二维码区域 | 显示该 URL 的二维码（若有此功能） | ☐ |
| 9.7 | 时间戳调整 | 对某时间戳使用「调整」或偏移（若有） | 显示调整后的时间或时间戳 | ☐ |

## 十一、异常与边界

| 用例编号 | 测试项 | 操作步骤 | 预期结果 | 通过 |
|---------|--------|----------|----------|------|
